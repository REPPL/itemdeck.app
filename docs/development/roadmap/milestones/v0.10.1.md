# v0.10.1 - UI Polish & Data Model Refinement

## Overview

Bug-fix and polish release addressing UI issues discovered after v0.10.0 (Data Editing). Focused on visual consistency, keyboard interaction fixes, and data model simplification.

## Bug Fixes

| Issue | Description | Solution |
|-------|-------------|----------|
| Gallery Rounded Corners | Images showed artefacts outside rounded container corners | `border-radius: inherit` in gallery CSS |
| Spacebar in EditForm | Users couldn't type spaces in textarea fields | Stop keyboard event propagation in overlay |
| Image Source Pluralisation | Showed "Image Source" with multiple images | Dynamic pluralisation based on count |
| Verdict Text Overflow | Long verdict text overlapped info section | CSS line clamping (2 lines max) |
| Raw Edit Metadata | Showed `EDITED AT: 1766751198970` in More overlay | Added fields to SKIP_FIELDS |

## Data Model Refinement

Simplified edit tracking from two fields to one:

**Before:**
```typescript
{ _hasEdits: true, _editedAt: 1735234567890 }
```

**After:**
```typescript
{ _editedAt: 1735234567890 }  // Presence implies hasEdits
```

## Dependencies

- **v0.10.0**: Data Editing (prerequisite)

## Key Implementation Notes

1. **CSS Inheritance**: Use `border-radius: inherit` for nested elements
2. **React Portal Events**: Portals maintain event bubbling through component tree
3. **Single-Purpose Fields**: Avoid redundant boolean flags when timestamp presence suffices
