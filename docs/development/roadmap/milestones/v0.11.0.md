## Key Implementation Notes

1. **Registry Pattern**: Central `MechanicRegistry` manages mechanic factories
2. **Lazy Loading**: Mechanics loaded on-demand via dynamic imports
3. **Mutual Exclusivity**: Only one mechanic active at a time
4. **Isolated Stores**: Each mechanic has its own Zustand store
