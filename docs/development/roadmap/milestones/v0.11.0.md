## Key Implementation Notes

### Mechanics Foundation

1. **Registry Pattern**: Central `MechanicRegistry` manages mechanic factories
2. **Lazy Loading**: Mechanics loaded on-demand via dynamic imports
3. **Mutual Exclusivity**: Only one mechanic active at a time
4. **Isolated Stores**: Each mechanic has its own Zustand store

### Discovery & Organisation

1. **Floating Search Bar**: Always visible above grid, keyboard shortcut `/`
2. **Filter Chips**: Dynamic chips for active filters (platform, year, genres)
3. **Grouping**: Collapsible sections by attribute with card counts
4. **View Modes**: Grid (current), List (compact rows), Compact (thumbnails)
5. **Statistics Dashboard**: CSS-only charts, enhances existing F-062
