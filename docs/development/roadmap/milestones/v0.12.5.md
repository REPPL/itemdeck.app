## Dynamic Field Discovery

### Remove Static Field Options

- [ ] Remove `SORT_FIELD_OPTIONS` from `fieldPathResolver.ts`
- [ ] Remove `SUBTITLE_FIELD_OPTIONS`, `FOOTER_BADGE_FIELD_OPTIONS`, etc.
- [ ] Update all consumers to use dynamic options only

### Schema-Driven Discovery

- [ ] Update `fieldDiscovery.ts` to be sole source of field options
- [ ] Improve field type inference from schema definitions
- [ ] Add support for nested field paths in schema

### Field Context Hints

```json
{
  "definition": {
    "fieldHints": {
      "sort": ["title", "year", "platform.title"],
      "badge": ["year", "categoryShort"],
      "subtitle": ["year", "status"],
      "groupBy": ["categoryTitle", "year", "decade"]
    }
  }
}
```

## Additional Features

### Nested Collection Paths

Support nested paths in MyPlausibleMe repositories:
```
data/collections/retro/gaming/
data/collections/retro/commercials/
```

### Settings Menu Reorganisation

Move UI Visibility to Appearance, consolidate Data tab into System.

### Smart Collection Caching

Skip re-caching when collection hasn't changed (use Last-Modified/ETag headers).
