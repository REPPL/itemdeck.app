# v1.0.0 - First Production Release

## Overview

v1.0.0 marks the first production-ready release of itemdeck.app. This milestone delivers a complete, polished experience with multi-collection comparison, reorganised settings, and consistent gaming mechanics UX.

**Theme:** Production Readiness, User Experience, Multi-Collection Support

## Goals

1. **Production Quality** - Stable, performant, accessible application
2. **User Transparency** - Clear navigation, discoverable settings
3. **Multi-Collection** - Compare and analyse across collections
4. **Gaming Excellence** - Consistent, discoverable mechanics experience
5. **Plugin Foundation** - Validated architecture for v1.5.0 ecosystem

## Key Implementation Notes

### F-064: Collection Comparison (as Mechanic)

Implement comparison as a mechanic rather than core navigation:
- Reuses existing mechanic infrastructure
- Accessed via Mechanics menu / Play settings tab
- Clean activation/deactivation lifecycle
- Display preferences for split view

See [F-064 Specification](../features/planned/F-064-collection-comparison.md) for full details.

### F-116: Settings Reorganisation

```
Current: Quick | Appearance | Collections | Data | System
    â†“
New:     Display | Collection | Play | System
```

The new "Play" tab centralises all mechanic settings and provides the entry point for game mechanics.

### F-037: Card Sorting Review

The v0.15.5 ViewPopover has a Sort/Group By interaction bug:
- Selecting "By Rank" + "Year" group displays cards sorted by year, ignoring rank
- Need to fix the interaction logic so sorting works within groups

Additionally, review invalid option combinations:
- Should Sort options change when a Group By is selected?
- Should some combinations be disabled or hidden entirely?
- Reference R-017 research for UX guidance

### F-118: Mechanics UX Standards

All mechanics MUST:
1. Have exit button in top-left position
2. Support Escape key to exit
3. Declare requirements in manifest
4. Show requirements in Play tab before activation
5. Have accessible help/instructions
